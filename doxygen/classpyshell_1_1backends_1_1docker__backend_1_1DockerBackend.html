Initializes the backend.

Parameters
pyshellThe pyshell instance to use when running docker commands to set up the backend. This must be an instance that is configured with a backend that will execute commands on the host system, e.g. the NativeBackend class.
imageThe docker image to use.
container_nameThe name of the container to use. If None, a random name will be generated.
portsA list of ports to expose. Each string in this argument will be passed to the --publish option of the docker run command.
use_sudoWhether to use sudo when running docker commands.
quietHow docker output from commands run by the backend should be handled. This will only apply to the commands run automatically by the backend to set up the container. The commands run by the user are not affected by this argument.
Exceptions
RuntimeErrorThrown if docker is not available or the image could not be pulled.
